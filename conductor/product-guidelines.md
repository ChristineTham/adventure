# Product Guidelines: Open Adventure Web

## Prose and Voice
- **Classic & Literal:** The game's narrative text, help messages, and documentation must strictly maintain the dry, informative style of the original 1970s game.
- **Lexicon:** Use Australian English spelling (e.g., 'colour', 'optimise').

## UX Principles
- **Modern Hybrid:** Retain the text-based core, but provide modern conveniences. Players can type commands OR click UI buttons generated by shadcn/ui to interact with objects and navigate.
- **Accessibility:** Ensure the web interface is fully accessible via screen readers, keyboard navigation, and respects contrast and focus states.
- **Instant Response:** The game state should update instantly without network round-trips during gameplay, simulating the fast response of the original terminal game.

## Branding and Visuals
- **Clean & Modern Retro:** Use a clean, modern design system (Tailwind v4 + shadcn/ui) that evokes nostalgia without resorting to forced "retro" fonts or scanlines unless requested as a theme.
- **Visual Consistency:** Maintain consistency with the original SVG and PNG assets where applicable, integrating them smoothly into the React UI.
